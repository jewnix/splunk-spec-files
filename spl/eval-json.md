# Evaluation Functions - JSON

Category: **Evaluation - JSON**  
Reference: [help.splunk.com](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions)  
Splunk version: 10.2

| Function | Syntax | Description | Usable With | Docs |
|----------|--------|-------------|-------------|------|
| `json_object` | `json_object(<members>)` | Creates a new JSON object from members of key-value pairs. | eval, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title2) |
| `json` | `json(<value>)` | Evaluates whether a value can be parsed as JSON. If the value is in a valid JSON format, the function returns the value. Otherwise, the function returns null. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title3) |
| `json_append` | `json_append(<json>, <path_value_pairs>)` | This function appends values to the ends of indicated arrays within a JSON document. This function provides a JSON eval function equivalent to the multivalue mvappend function. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title4) |
| `json_array` | `json_array(<values>)` | Creates a JSON array using a list of values. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title5) |
| `json_array_to_mv` | `json_array_to_mv(<json_array>, <boolean>)` | This function maps the elements of a proper JSON array into a multivalue field. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title6) |
| `json_delete` | `json_delete(<object>,<keys>)` | Use json_delete to remove one or more keys and their corresponding values from the specified JSON object. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title7) |
| `json_entries` | `json_entries(<value>)` | Returns the key-value entries from the top-level key-value pairs in a JSON object. The entries are returned as a JSON array of JSON objects with fields key and value . | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title8) |
| `json_extend` | `json_extend(<json>, <path_value_pairs>)` | Use json_extend when you want to append multiple values at once to an array. json_extend flattens arrays into their component values and appends those values to the ends of indicated arrays within a valid JSON document. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title9) |
| `json_extract` | `json_extract(<json>, <paths>)` | This function returns a value from a piece of JSON and zero or more paths. The value is returned in either a JSON array, or a Splunk software native type value. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title10) |
| `json_extract_exact` | `json_extract_exact(<json>, <keys>)` | Like the json_extract function, this function returns a Splunk software native type value from a piece of JSON. The main difference between these functions is that the json_extract_exact function does not use paths to locate and extract values, but instead matches literal strings in the event and extracts those strings as keys. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title11) |
| `json_has_key_exact` | `json_has_key_exact(<object>, <key>)` | This function evaluates whether a JSON object contains the specified key and returns either TRUE or FALSE. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title12) |
| `json_keys` | `json_keys(<json>)` | Returns the keys from the key-value pairs in a JSON object. The keys are returned as a JSON array. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title13) |
| `json_set` | `json_set(<json>, <path_value_pairs>)` | Inserts or overwrites values for a JSON node with the values provided and returns an updated JSON object. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title14) |
| `json_set_exact` | `json_set_exact(<json>, <key_value_pairs>)` | Generates or overwrites a JSON object using the key-value pairs that you specify. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title15) |
| `json_valid` | `json_valid(<json>)` | Evaluates whether a piece of JSON uses valid JSON syntax and returns either TRUE or FALSE. | eval, fieldformat, where | [docs](https://help.splunk.com/en/splunk-enterprise/spl-search-reference/10.2/evaluation-functions/json-functions#ariaid-title16) |
